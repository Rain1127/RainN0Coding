# 前端页面描述

## 1. RAG 智能问答首页

该页面是 RAG 智能问答系统的首页，整体采用左侧侧边栏和右侧主内容区的双栏布局。页面以浅色背景为主，主区域留白充足，视觉焦点集中在中间的大型提问框。

左侧侧边栏顶部展示应用图标、标题“RAG 智能问答”和副标题“Powered by AI”。其下是“快速开始”卡片，卡片中包含“新建对话”入口，左侧是蓝色加号按钮，右侧显示“新建对话 / 从空白开始”。卡片底部还有“管理后台”按钮，带齿轮图标，用于进入后台管理界面。

侧边栏中部是“搜索对话”区域，包含搜索输入框和快捷键提示“Ctrl / Cmd + K”。下方是历史会话列表，按“今天”“7天内”等时间分组展示，会话以文本标题形式纵向排列。侧边栏底部展示当前用户 admin、头像以及更多操作按钮。

右侧主内容区顶部左侧显示当前会话标题“新对话”，右上角有“Star”按钮和更多菜单按钮。页面中央展示小号品牌标识“RAG 智能问答”，主标题为“把问题变成清晰答案”，其中“清晰答案”使用蓝色强调。标题下方有说明文案，表达系统可结合机构知识、知识检索和深度思考生成可执行方案。

主标题下方是核心输入区。输入框为大尺寸圆角框，提示文案为“输入需要深度分析的问题...”，右侧有发送图标按钮。输入框左下方提供“深度思考”按钮，用于开启或标识深度推理模式。输入框下方还显示快捷键说明：“Enter 发送，Shift + Enter 换行”。

页面底部中间有推荐问题区域，标题类似“试试这些开始”，下面横向排列三张推荐卡片。卡片分别对应“实时数据”“系统交互”“业务系统”等场景，每张卡片包含图标、标题、示例问题和右侧跳转箭头，可用于快速发起示例提问。

## 2. RAG 智能问答详情页

该页面展示某次问答会话的回答内容，布局延续首页的左侧会话栏和右侧内容区结构。左侧侧边栏中，“销售数据统计内容”被蓝色背景高亮，表示当前选中的会话，右侧带有三点更多操作按钮。

主内容区顶部显示当前会话标题“销售数据统计内容”，右上角保留“Star”和更多菜单按钮。内容区域以文档阅读形式排版，包含表格、提示块、分级标题、项目符号列表和结论文本。

页面上方可见一张销售区域统计表，表头包含“地区”“销售额”“占比”。表格列出华南、华东、西北、西南、华北等地区，并分别展示金额和占比。表格下方有浅蓝色提示块，左侧带蓝色竖线，内容说明地区分类不包含部分区域，并提示占比合计可能存在四舍五入偏差。

回答正文继续展示“销售趋势”“销售预测”等章节。章节标题使用较粗字体，正文通过项目符号列出分析依据。内容指出当前信息中缺少历史月份数据，因此无法计算环比、同比、趋势线、季节性特征或预测结果。

回答结束位置有一排反馈和操作图标，包括复制、点赞、点踩等按钮。页面底部是输入区域，输入框提示“输入你的问题...”，左下方有“深度思考”按钮，右侧有发送按钮。输入框下方继续显示“Enter 发送，Shift + Enter 换行”的快捷键说明。右侧滚动条表示回答内容支持纵向滚动。

## 3. 管理后台 - 知识库管理页

该页面是 Ragent 管理后台的知识库管理列表页，采用深色左侧导航栏和浅灰色主内容区的后台布局。左侧导航栏顶部显示紫色 R 图标、标题“Ragent 管理后台”和副标题“Knowledge Console”。

左侧导航分为“导航”和“设置”两组。导航菜单包含“知识库管理”“意图树配置”“数据通道”“流水线管理”“流水线任务”等；设置菜单包含“用户管理”“示例问题”“系统设置”等。当前“知识库管理”菜单以高亮背景显示。侧边栏底部有“收起侧边栏”按钮。

顶部横向栏左侧是全局筛选输入框，提示“筛选知识库...”，右侧显示快捷键“Ctrl K”。顶部右侧依次有“返回聊天”按钮、“Star”按钮、更多按钮，以及当前用户 admin 的头像和下拉入口。

主内容区顶部通过面包屑显示“首页 / 知识库管理”。页面标题为“知识库管理”，副标题为“管理所有知识库及其文档”。标题右侧是操作区，包括“搜索知识库名称”输入框、“搜索”按钮、“刷新”按钮和紫色主按钮“新建知识库”。

标题下方横向排列四张统计卡片，分别展示知识库数量、文档数、含文档知识库数量和创建用户数。每张卡片左侧有淡紫色圆形图标，中间显示指标名称和数字，右侧有“全部”标签。

下方是知识库表格。表头包括“名称”“Embedding模型”“Collection”“文档数”“负责人”“创建时间”“修改时间”“操作”。知识库名称以蓝色链接显示，Collection 以圆角标签展示。操作列提供“编辑”和“删除”按钮，并分别配有铅笔和垃圾桶图标。表格底部左侧显示“共 6 条”，右侧是分页控件，包括“上一页”“1 / 1”“下一页”。

## 4. 管理后台 - 文档管理页

该页面用于管理某个知识库下的文档。整体结构与后台其他页面一致：左侧为深色导航栏，顶部为全局操作栏，右侧为主内容区。当前仍处于“知识库管理”模块下。

主内容区顶部面包屑为“首页 / 知识库管理 / 文档管理”。页面标题是“文档管理”，副标题显示当前知识库“测试知识库3（testgroup）”。右上角提供“返回知识库”按钮和紫色主按钮“上传文档”，分别用于返回知识库列表和新增文档。

页面主体是白色卡片区域，标题为“文档列表”，副标题为“支持筛选与分块管理”。卡片右上角提供筛选工具，包括“搜索文档名称”输入框、“搜索”按钮、“全部状态”下拉框和“刷新”按钮。

文档表格包含“文档”“来源”“处理模式”“状态”“启用”“分块数”“类型”“大小”“更新时间”“操作”等列。文档列左侧有文件夹图标，文件名以蓝色链接显示。来源显示为 Local File，处理模式为 chunk。状态列使用绿色圆点和 success 文本表示处理成功。启用列使用蓝色开关控件表示文档是否启用。

操作列以图标按钮形式展示，包含编辑、查看或执行、统计或详情、删除等操作。表格底部左侧显示“共 3 条”，右侧提供“上一页”“1 / 1”“下一页”的分页控件。

## 5. 管理后台 - 分块管理页

该页面用于管理文档切分后的 Chunk 数据。左侧和顶部导航与管理后台保持一致。主内容区顶部面包屑显示“首页 / 知识库管理 / 文档管理 / 切片管理”。页面标题为“分块管理”，副标题显示当前文档“基本信息.md”和对应知识库编号。

页面右上角有三个主要按钮：“返回文档”“重建向量”和紫色主按钮“新建分块”。“返回文档”用于回到文档列表，“重建向量”用于重新生成向量索引，“新建分块”用于手动新增 Chunk。

主体卡片标题为“Chunk 列表”，副标题为“支持编辑、启停、批量操作”。卡片右上角包含“全部状态”下拉筛选、“刷新”按钮、“批量启用”“批量禁用”“全量启用”“全量禁用”等批量操作按钮。表格首列为复选框，用于选择单条或多条分块。

Chunk 表格包含“序号”“内容”“状态”“字符数”“Token数”“更新时间”“操作”等列。内容列展示分块文本摘要，超出部分以省略号截断。状态列使用圆角标签显示“启用”。字符数和 Token 数用于展示分块长度。每行右侧操作区包含“编辑”“禁用”“删除”按钮，用于修改内容、切换启用状态或移除分块。

表格底部左侧显示“共 4 条”，右侧为分页控件，包含“上一页”“1 / 1”“下一页”。

## 6. 管理后台 - 意图树配置页

该页面用于配置问答系统的意图图层级、类型和节点关系。左侧导航栏中“意图树配置”菜单高亮。顶部栏仍包含知识库筛选输入框、“返回聊天”“Star”按钮和用户信息区。

主内容区顶部面包屑显示“首页 / 意图树配置”。页面标题为“意图树配置”，副标题为“配置意图层级、类型和节点关系”。右上角有“刷新”按钮和紫色主按钮“新建根节点”。

页面主体采用左右双栏布局。左侧卡片标题为“意图树结构”，提示“点击节点查看详情或进行编辑”。树形结构通过缩进展示层级关系，节点前有展开或收起箭头，节点名称后方带类型标签和来源标签，例如 DOMAIN、CATEGORY、TOPIC、KB、MCP 等。

当前选中的根节点“集团信息化”以浅蓝色背景高亮。其下包含“公司网络”“人事”“IT支持”“财务”“业务系统”“销售汇总数据统计”等节点。部分节点展开后展示更深层级，如“OA系统”“保险系统”“系统介绍”“架构设计”“数据安全”等。

右侧卡片标题为“节点详情”，用于展示和管理当前选中节点。顶部显示节点名称“集团信息化”，旁边有 DOMAIN、KB、启用等标签。右侧提供“新建子节点”“编辑节点”“删除节点”三个操作入口，其中“新建子节点”为紫色主按钮。

详情区域以键值形式展示父节点、排序、Collection 等字段，并包含“描述”和“示例问题”两个内容区。当前描述显示“暂无描述”，示例问题显示“暂无示例”。页面右侧有滚动条，说明意图树内容可纵向滚动。

## 7. 管理后台 - 任务详情弹窗

该页面展示任务详情弹窗。背景页面被半透明深色遮罩覆盖并模糊处理，突出中央的白色弹窗。弹窗尺寸较大，采用圆角设计，右上角有关闭按钮“×”。

弹窗标题为“任务详情”，下方显示任务 ID。任务状态使用紫色胶囊标签展示为 completed。基础信息区域分为左右两列：左侧展示 Pipeline ID、Source 类型、来源 URL 和 Chunks 数量；右侧展示 Created、Started、Completed 等时间字段。

中部是“任务元数据”区域，以浅灰色代码块形式展示 JSON 内容，当前显示为空对象 `{}`。下方是“节点执行日志”表格，表头包括“节点”“类型”“状态”“耗时”“消息”。

日志表格中的每一行代表流水线中的一个处理节点，例如 fetcher-1、parser-1、enhancer-1、chunker-1、indexer-1。类型列显示 fetcher、parser、enhancer、chunker、indexer。状态列均使用紫色 success 标签。耗时列展示毫秒级执行时间，消息列说明各节点执行结果，例如已获取字节数、解析文本长度、增强完成、已分块、已写入集合等。

该弹窗主要用于查看流水线任务执行过程和排查处理结果。用户可以通过右上角关闭按钮退出弹窗，返回被遮罩的后台页面。
