# 鱼AI代码助手 — 完整后端调用链

## 架构总览

```
前端 (Vue3 + Vite :5173)
    │
    │ HTTP SSE / JSON
    ▼
Java Spring Boot (:8123/api)
    │
    │ WebClient (HTTP)
    ▼
Python FastAPI (:8000/api)
    │
    │ LangChain + OpenAI SDK
    ▼
DeepSeek API (api.deepseek.com)
    │
    │ 回退: DashScope (阿里云)
    ▼
    ──► MySQL (业务数据)

## 一、主流程：AI 对话生成代码

这是最核心的调用链，用户在前端输入需求 → 经历 Java/Python/AI → 最终拿到生成的代码。

### 第 1 步：前端发起请求

```
GET /api/app/chat/gen/code?appId=123&message=帮我生成一个登录页面
Accept: text/event-stream
```

使用原生 `fetch`（不走 axios），建立 SSE 长连接。

### 第 2 步：Java Controller 层

**文件**: `controller/AppController.java:72-102`

```
AppController.chatToGenCode(appId, message, request)
│
├─ 1. 参数校验：appId > 0，message 非空
├─ 2. 限流检查：@RateLimit，每用户 60 秒最多 5 次（Redisson 令牌桶）
├─ 3. 获取当前登录用户：userService.getLoginUser(request)
│         └─ 从 HttpSession 获取 User → 数据库刷新 → 返回 User 对象
├─ 4. 调用 appService.chatToGenCode(appId, message, loginUser)
│         └─ 返回 Flux<String>（响应式流，非阻塞）
└─ 5. 将 Flux<String> 包装为 Flux<ServerSentEvent>
      └─ 每个 chunk 包装为 {"d": chunk}
      └─ 末尾追加 event:done
```

### 第 8 步：LLM 调用与容错

model_router.call_with_fallback()
│

│     ├─ 推理组: deepseek-v4-pro → deepseek-chat → qwen3.6-flash
│     ├─ 结构化组: deepseek-chat → qwen3.6-flash
│     └─ 轻量级组: deepseek-chat → qwen3.6-flash
│
├─ 逐候选尝试:
│     │
│     ├─ 检查熔断器状态
│     │     ├─ CLOSED: 正常调用
│     │     ├─ OPEN (3次连续失败, 30秒冷却): 跳过此候选
│     │     └─ HALF_OPEN: 尝试1次探测调用
│     │
│     ├─ 调用成功 → 熔断器关闭
│     │
│     └─ 调用失败 → 熔断器计数+1 → 尝试下一个候选
│
└─ 所有候选失败 → 返回错误给调用方
```

## 六、关键容错机制

| 层级 | 机制 | 说明 |
|------|------|------|
| 前端 | fetch AbortController | 用户可手动中止 SSE 流 |
| Java | @RateLimit + Redisson | 每用户 60s/5次 限流 |
| Java | 虚拟线程 | chat/截图操作异步不阻塞 |
| Python | 熔断器 CircuitBreaker | 模型连续 3 次失败 → 冷却 30s |
| Python | 模型回退 Router | 主模型失败 → 自动切换备用模型 |
| Python | 审查重试 | 代码评分 <80 分 → 重试(最多3次) |
| Python | 会话记忆 fallback | Redis 不可用 → 内存字典 |
