# RainN0Coding 前端重建设计

## 背景

当前主分支没有可维护的 Vue 前端源码，只保留一套中文乱码且功能有限的静态页面。`codex/bug` 分支包含登录、注册、聊天和管理后台的 Vite 编译产物；隐藏 checkpoint `276def757ffbac589b8dc62e4834cbe93b5c3cae` 保存了这些产物对应的完整 Vue 3 源工程。

本次工作以 checkpoint 中的业务逻辑和接口封装为参考，在仓库约定的 `RainN0Coding-frontend/` 目录中重建工程。目标不是逐像素复原旧界面，而是在保持后端接口兼容的前提下，交付一套可维护、完整、响应式且符合无障碍规范的前端。

## 目标与非目标

### 目标

- 恢复 Vue 3、TypeScript、Vite、Pinia、Vue Router 和 Ant Design Vue 工程。
- 完成登录、注册、生成首页、项目工作台、项目管理和管理后台。
- 正确接入当前 Java API 与 Python SSE 事件格式。
- 使用“明亮开发者工作台”视觉方向统一所有页面。
- 提供明确的加载、空、成功、错误、无权限和未找到状态。
- 通过 TypeScript、生产构建、关键单元测试和浏览器关键流程检查。

### 非目标

- 不修改 Java、Python、数据库或 AI 工作流行为。
- 不新增知识库、计费、团队协作等后端未支持的产品能力。
- 不反向维护 `codex/bug` 中的压缩构建文件。
- 不把技术日志直接混入主要聊天阅读流。

## 信息架构

### 访客页面

- `/login`：账号密码登录，支持字段校验、密码显隐、提交中状态与错误恢复。
- `/register`：账号注册、密码强度提示、确认密码校验和注册成功反馈。

### 登录用户页面

- `/`：生成首页。包含产品介绍、需求输入、示例提示、最近项目和服务状态摘要。
- `/chat/:appId`：项目工作台。包含项目对话、生成阶段、文件列表、下载、部署和预览入口。
- `/projects`：我的项目。支持搜索、状态筛选、分页、打开项目、删除确认和空状态。

### 管理员页面

- `/admin/apps`：应用管理。
- `/admin/apps/:appId`：应用详情与管理操作。
- `/admin/users`：用户管理。
- `/admin/intent-tree`：意图树查看、编辑、保存和重置。

### 通用页面

- `/403`：无权限。
- 通配路由：未找到页面。
- 网络异常、服务不可用和数据为空采用可复用的页面内状态组件。

## 页面与组件边界

### 布局

- `AppShell`：登录后通用外壳，负责桌面侧栏、移动端抽屉、顶部工具区和主内容跳转入口。
- `AuthLayout`：登录与注册共享的品牌和表单布局。
- `AdminLayout`：管理员导航和内容容器。
- 项目工作台在桌面端使用主内容加右侧上下文栏；小屏下按主内容、进度、文件的优先级纵向排列。

### 共享组件

- `BrandMark`、`UserMenu`、`PageHeader`、`StatusBadge`。
- `LoadingState`、`EmptyState`、`ErrorState`、`ConfirmAction`。
- `PromptComposer`：需求输入、示例提示、快捷键和提交状态。
- `AgentProgress`：展示 Intent、PM、Architect、Coder、Reviewer、Builder 阶段。
- `GeneratedFileList`：文件路径、语言和生成状态。
- `ProjectCard` 与 `ProjectListToolbar`：项目列表展示与筛选。

组件只通过明确的 props、事件或 store action 通信；页面负责数据编排，展示组件不直接调用 API。

## 状态与数据流

Pinia 按职责拆分为：

- `auth`：当前用户、初始化状态、登录、注册和退出。
- `apps`：项目列表、筛选、分页和当前项目摘要。
- `generation`：SSE 连接、阶段事件、生成文件、流状态和错误。
- `ui`：侧栏、移动端抽屉和非业务界面状态。

普通请求统一经过 Axios 客户端：

1. 使用 `/api` 基础路径和 Cookie 会话。
2. 解包 `BaseResponse`，业务 `code !== 0` 视为失败。
3. 401 清理本地登录态，并跳转 `/login?redirect=<原路径>`。
4. 超时、离线和后端错误生成可恢复的用户提示。

生成流程为：

1. `POST /app/add` 创建应用并获取 `appId`。
2. 打开 `GET /app/chat/gen/code` SSE 连接。
3. 解析标准 `data: { ... }` JSON 事件。
4. 根据 `phase`、`type`、`message`、`file_path` 等字段更新阶段、日志和文件。
5. 对 `error` 与非成功 `done` 事件进入失败状态；正常结束进入成功状态。
6. 完成后刷新应用详情和项目列表。

SSE 解析器必须独立于 Vue 组件并可单元测试。它需要处理分块边界、多个 frame、空行、嵌套字符串数据、未知事件和不完整尾帧。用户可以取消正在进行的生成；中断时保留已收到的内容与文件。

## 错误处理与反馈

- 表单错误显示在对应字段下方，不能只依赖全局消息。
- 请求错误显示在关联内容区域，并提供重试或返回入口。
- 删除等破坏性动作必须二次确认。
- 部署成功展示部署地址，由用户主动选择打开，避免弹窗被浏览器拦截。
- 超过约 300ms 的异步加载显示骨架或进度反馈。
- SSE 失败不清空现有消息、阶段或文件；重试动作使用相同项目继续。
- 空项目、空历史、无搜索结果和服务不可用均有专用文案与下一步操作。

## 视觉系统

采用已确认的方案 A“明亮开发者工作台”：

- 深色石板侧栏、浅灰工作区、白色内容表面。
- 靛蓝为主操作色，绿色只表示成功，红色只表示错误或危险。
- 正文使用 IBM Plex Sans 风格系统字体栈；代码、文件路径、事件时间使用 JetBrains Mono 风格等宽字体栈。
- Ant Design Vue `ConfigProvider` 令牌统一颜色、字号、圆角、控件高度和焦点环。
- 卡片使用轻边框、极弱阴影和 12–16px 圆角，避免高饱和渐变和装饰性玻璃效果。
- 图标统一使用 Ant Design 线性图标，不使用 Emoji 充当结构图标。
- 微交互时长为 150–200ms，只使用 opacity 和 transform，并尊重 `prefers-reduced-motion`。

## 响应式与无障碍

- 在 375、768、1024、1440px 宽度验证布局。
- 桌面侧栏在小屏折叠为顶部栏和抽屉，主内容不可产生横向滚动。
- 所有点击目标至少 44×44px，图标按钮提供 `aria-label`。
- 页面使用 `header`、`nav`、`main`、`section`、`button` 等语义元素。
- 提供“跳到主要内容”链接，标题层级连续，键盘焦点清晰可见。
- 弹窗和抽屉支持 Escape 关闭，并在关闭后恢复触发元素焦点。
- 状态信息不只依赖颜色，还包含图标或文字。
- 表单具有可见标签、正确的 autocomplete 和相关错误描述。

## 测试与验收

### 自动化测试

- SSE 解析器：分块、多个事件、异常 JSON、完成、错误和中断场景。
- `auth` store：初始化、登录、退出和失败状态。
- 路由守卫：访客重定向、登录后回跳、管理员权限和 403。
- 表单：必填、密码一致性和提交中禁用。
- 关键状态组件：加载、空、错误和重试事件。

### 构建与浏览器验证

- 执行 `npm run test`、`npm run build`。
- 验证登录、注册跳转、受保护路由、生成首页、项目工作台和管理员守卫。
- 验证 375px 与桌面布局、键盘导航、焦点可见性和 reduced-motion。
- 按最新 Web Interface Guidelines 复核前端源码，并修复阻断级和高优先级问题。

## 迁移策略

1. 从 checkpoint 恢复除 `node_modules`、`dist` 外的前端源码到 `RainN0Coding-frontend/`。
2. 先以测试锁定认证、路由和 SSE 行为，再重构实现。
3. 建立主题令牌、共享状态组件和布局。
4. 逐页重做访客、用户和管理员界面。
5. 生产构建输出到 Spring Boot 静态资源目录，替换乱码占位构建产物。

恢复操作只新增前端工程并更新其构建输出，不覆盖当前工作区中与本任务无关的用户修改。
